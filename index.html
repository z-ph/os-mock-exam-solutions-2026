操作系统模拟试卷（一）答案与过程

根据用户提供的《操作系统模拟试卷一(1).pdf》逐题作答。概念题按关键词整理，计算题保留公式、步骤和结果。

第 1 题：进程、线程、PCB、进程状态转换

题干

回答下列问题：

  1. 为什么操作系统中要引入“进程”概念？
  2. 进程由哪几部分组成？PCB 的作用是什么？
  3. PCB 中通常包含哪些信息？至少写出四类。
  4. 什么是线程？线程和进程的主要区别是什么？
  5. 画出或用文字说明进程的三种基本状态及其转换原因。
考点：为什么引入进程、进程组成、PCB 作用和内容、线程概念、三态转换。
  1. 引入进程是为了描述程序的并发执行，使操作系统能够对并发活动进行调度、分配资源、保护现场和控制执行顺序。程序是静态的指令集合，进程是程序在某个数据集合上的一次动态执行过程。
  2. 进程通常由程序段、数据段和进程控制块 PCB 组成。PCB 是操作系统感知、管理和控制进程的主要数据结构，可以看作进程存在的标志。
  3. PCB 中通常包含：进程标识符 PID、进程当前状态、CPU 现场信息、优先级、程序和数据地址、阻塞原因、同步机制、资源清单、队列链接指针等。写出其中四类即可。
  4. 线程是进程内一个相对独立的执行流，是 CPU 调度和执行的基本单位。进程是资源分配的基本单位；同一进程内的线程共享进程资源，如地址空间、打开文件等。
  5. 三种基本状态：就绪、执行、阻塞。就绪经进程调度变为执行；执行因时间片用完变为就绪；执行因等待 I/O 或事件变为阻塞；阻塞因 I/O 完成或等待事件发生变为就绪。

第 2 题：分页、分段、段页式、快表、动态重定位

题干

回答下列问题：

  1. 什么是分页存储管理？逻辑地址由哪两部分组成？
  2. 页表的作用是什么？页表项中通常包含哪些内容？
  3. 什么是快表？快表如何提高地址变换速度？
  4. 分页和分段的主要区别是什么？
  5. 什么是段页式存储管理？其地址结构是什么？
  6. 什么是动态重定位？写出动态重定位下物理地址的计算公式。
  1. 分页存储管理把进程逻辑空间划分为大小相等的页，把内存划分为大小相等的物理块，页可以离散装入物理块。逻辑地址由页号和页内位移组成。
  2. 页表完成页号到物理块号的映射。页表项通常包含物理块号、状态位/存在位、访问权限、修改位、访问位等。
  3. 快表 TLB 是页表项的高速缓存。若页号在快表中命中，可直接得到物理块号，减少访问内存页表的次数，从而提高地址变换速度。
  4. 分页与分段区别：页是物理单位，段是逻辑单位；页大小固定，段长不固定；分页地址是一维地址经系统划分页号，分段地址通常为段号+段内地址；分页主要提高内存利用率，分段便于共享、保护和动态链接。
  5. 段页式存储管理先按逻辑意义分段，再把每段划分页。地址结构通常为：段号 S、段内页号 P、页内位移 W。
  6. 动态重定位是在程序运行时把逻辑地址转换成物理地址，通常依赖重定位寄存器等硬件支持。
动态重定位： 物理地址 = 重定位寄存器内容 + 逻辑地址

第 3 题：I/O 控制方式、SPOOLing、设备独立性

题干

回答下列问题：

  1. 操作系统中常见的 I/O 控制方式有哪四种？
  2. 分别说明程序直接控制方式、中断驱动方式、DMA 方式、通道方式的工作特点。
  3. 什么是 SPOOLing 系统？它为什么能把独占设备改造成共享设备？
  4. SPOOLing 系统通常由哪几部分组成？
  5. 什么是设备独立性？它有什么优点？
  1. 常见 I/O 控制方式有四种：程序直接控制方式、中断驱动方式、DMA 方式、I/O 通道方式。
  2. 程序直接控制方式：CPU 发出 I/O 命令后不断查询设备状态，CPU 干预最多，效率低。中断驱动方式：CPU 发命令后可执行其他任务，设备完成后用中断通知 CPU。DMA 方式：CPU 设置 DMA 参数后，由 DMA 控制器直接完成设备与内存之间的数据块传送，结束后中断 CPU。通道方式：通道类似专门的 I/O 处理机，可执行通道程序，控制多个设备或控制器完成 I/O。
  3. SPOOLing 是假脱机技术，用磁盘和后台进程模拟脱机输入/输出，使外围操作与 CPU 处理并行。它把用户输出先放入磁盘队列，再由假脱机进程按顺序送给独占设备，因此能把独占设备在逻辑上改造成共享设备。
  4. SPOOLing 系统一般包括输入井、输出井、输入缓冲区、输出缓冲区、输入进程、输出进程。以打印为例，还包括磁盘缓冲区、打印缓冲区、假脱机管理进程和假脱机打印进程。
  5. 设备独立性是指用户程序使用逻辑设备名和统一接口，不直接依赖具体物理设备。优点是提高可移植性、灵活性和设备替换能力，也便于统一分配、保护、缓冲和差错控制。

第 4 题：文件目录、逻辑结构、物理结构、索引结点

题干

回答下列问题：

  1. 文件目录的作用是什么？什么是按名存取？
  2. 什么是 FCB？FCB 中通常包含哪些内容？
  3. 什么是索引结点 inode？采用索引结点有什么优点？
  4. 文件的逻辑结构主要有哪些类型？
  5. 文件的物理结构主要有哪些类型？分别说明其优缺点。
  1. 文件目录用于记录文件名与文件控制信息之间的对应关系，实现文件查找、创建、删除、保护和共享。按名存取是指用户只需给出文件名，系统通过目录找到文件的位置和控制信息。
  2. FCB 是文件控制块，通常包含文件名、文件类型、文件长度、物理位置、逻辑结构、物理结构、存取权限、创建/修改时间、拥有者等信息。
  3. inode 是索引结点，把文件属性和物理地址等控制信息从目录项中分离出来。采用 inode 后，目录项可只保存文件名和 inode 号，目录更小，检索时需要读入的目录块更少，也有利于文件共享。
  4. 文件逻辑结构主要有：顺序文件、索引文件、索引顺序文件、直接文件/散列文件等。考试中重点掌握顺序文件和索引文件。
  5. 文件物理结构主要有连续分配、链接分配、索引分配。连续分配访问快，支持顺序和随机访问，但容易产生外部碎片且扩展困难；链接分配便于扩展，不要求连续空间，但随机访问慢；索引分配支持随机访问，扩展能力好，但需要额外索引块开销。

题干

某分页存储管理系统中，页面大小为 1KB，某进程页表如下：

页号块号状态位
051
181
231
3-0
461
  1. 逻辑地址 2500 对应的页号和页内位移是多少？若合法，求物理地址。
  2. 逻辑地址 3500 对应的页号和页内位移是多少？是否会产生缺页中断？为什么？
  3. 逻辑地址 5000 是否越界？为什么？
  4. 若页面大小为 4KB，逻辑地址为十六进制 2A3BH，求其页号和页内位移。
  5. 若第 2H 页对应物理块号为 9H，则逻辑地址 2A3BH 对应的物理地址是多少？

页面大小 1KB = 1024B。公式：

页号 = INT(逻辑地址 / 页面大小) 页内位移 = 逻辑地址 mod 页面大小 物理地址 = 块号 * 页面大小 + 页内位移
  1. 逻辑地址 2500：页号 = 2500/1024 取整 = 2；页内位移 = 2500 mod 1024 = 452。页 2 状态位为 1，块号为 3，物理地址 = 3*1024+452 = 3524。
  2. 逻辑地址 3500：页号 = 3；页内位移 = 428。页 3 状态位为 0，表示该页不在内存，会产生缺页中断。
  3. 逻辑地址 5000：页号 = 4；页内位移 = 904。页表中存在页 4，且位移小于 1024，所以不越界。页 4 状态位为 1，若继续求物理地址，则为 6*1024+904 = 7048。
  4. 页面大小 4KB = 212 = 1000H。逻辑地址 2A3BH 的页号为高位 2H，页内位移为 A3BH。
  5. 若第 2H 页对应物理块号 9H，则物理地址 = 9H*1000H + A3BH = 9A3BH，即十进制 39483。

  3. 若要访问该文件的逻辑块号 3，应访问哪个物理块？

（二）混合索引

某文件系统采用混合索引方式。索引结点中有：10 个直接地址项；1 个一级间接地址项；1 个二级间接地址项；1 个三级间接地址项。磁盘块大小为 1KB，每个地址项占 4B。

  1. 一个索引块最多能存放多少个地址项？
  2. 该文件系统支持的最大文件块数是多少？
  3. 该文件系统支持的最大文件长度是多少？

（一）FAT 文件系统

  1. FAT 表项数 = 磁盘块数 = 4096，每项 16 bit，所以总位数 = 4096*16 = 65536 bit。
  2. 总字节数 = 65536/8 = 8192 Byte。
  3. 每块 1KB = 1024 Byte，FAT 占用磁盘块数 = 8192/1024 = 8 块。
  4. 文件链：6 → 11 → 20 → 35 → 50 → EOF，共占用 5 个磁盘块。
  5. 每块 1KB，因此该文件最大可存放 5*1024 = 5120 Byte 数据。
  6. 若逻辑块号从 0 开始编号：逻辑块 0、1、2、3 分别对应 6、11、20、35，所以逻辑块号 3 应访问物理块 35。
如果老师把“逻辑块号 3”口语化理解为“第 3 个逻辑块”，则对应物理块 20。标准系统题通常按 0 号开始编号。

（二）混合索引

  1. 磁盘块大小 1KB = 1024B，每个地址项 4B，所以一个索引块最多存放 1024/4 = 256 个地址项。
  2. 最大文件块数 = 10 个直接块 + 1 个一级间接块能指向的 256 块 + 1 个二级间接块能指向的 2562 块 + 1 个三级间接块能指向的 2563 块。
  3. 最大文件块数 = 10 + 256 + 65536 + 16777216 = 16843018 块。最大文件长度 = 16843018*1024 = 17247250432 Byte，约 16.06 GiB。
  4. 逻辑块号 300：直接地址覆盖 0-9；一级间接覆盖 10-265；300 超出一级间接范围，落入二级间接范围，所以通过二级间接索引访问。

第 10 题：死锁与资源分配图

题干

某系统中有 3 个进程 P1、P2、P3，有 3 类资源 R1、R2、R3，每类资源均只有 1 个实例。当前资源分配情况如下：

R1 已分配给 P1；R2 已分配给 P2；R3 已分配给 P3；P1 正在请求 R2；P2 正在请求 R3；P3 正在请求 R1。

回答下列问题：

  1. 画出该系统的资源分配图。
  2. 判断该系统是否发生死锁，并说明理由。

  4. 若要预防死锁，可以分别破坏哪些条件？请举例说明。
  5. 若每类资源都有多个实例，资源分配图中有环是否一定死锁？为什么？

也可写成环：P1 → R2 → P2 → R3 → P3 → R1 → P1。

  1. 该系统发生死锁。因为每类资源只有 1 个实例，资源分配图中存在环，且环中每个进程都占有一个资源并请求下一个资源，无法继续推进。
  2. 死锁四个必要条件：互斥条件、请求和保持条件、不可剥夺条件、循环等待条件。
  3. 预防死锁可以破坏四条件之一：破坏请求和保持，可要求进程一次性申请全部资源；破坏不可剥夺，可规定申请新资源失败时释放已占有资源；破坏循环等待，可给资源编号并按序申请。互斥通常是资源固有属性，很多设备不能破坏。
  4. 若每类资源有多个实例，资源分配图有环不一定死锁。此时有环只是可能死锁的必要条件，不是充分条件；因为环外或同类资源的其他实例可能使某些进程继续执行并释放资源。
